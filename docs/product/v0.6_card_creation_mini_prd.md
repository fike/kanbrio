**Status**: Proposal | **Version**: 0.6 | **Owner**: @product-manager | **Date**: 2026-05-30
**Strategic Alignment**: Immediate User Activation, Time-to-Value (TTV) Optimization, Core Flow Integrity, Low-Latency Micro-interactions, and Multi-tenant Security Isolation.

## ⚙️ 1. Numbered Functional Requirements (FR)

### 1.1 Inline Form Mounting & Accessibility (a11y)
*   **FR-50 (Inline Mount Transition)**:
    *   Triggering "+ Add Card" in any column/swimlane grid intersection must unmount the button and mount the card authoring form in the DOM within 150ms using a crossfade.
*   **FR-51 (Autofocus & Tab Loop)**:
    *   Upon mounting, the card title input textarea MUST programmatically receive focus within 50ms.
    *   Tabbing must loop focus within the active inline card: `Textarea` -> `Add Button` -> `Cancel Button` -> back to `Textarea`.
*   **FR-52 (Escape & Blur Dismissal)**:
    *   Pressing `Escape` or clicking completely outside the inline form card must immediately cancel the creation, unmount the form, and restore the "+ Add Card" trigger button, returning focus to it.

### 1.2 Form Validation & Error States
*   **FR-53 (Sanitization & Length Bounds)**:
    *   The frontend and backend must strip leading and trailing whitespace from the title. The title must contain between 1 and 255 characters. Empty submissions must be rejected on the client side.
*   **FR-54 (WIP Limit Interception & Shaking)**:
    *   If the backend returns a `409 Conflict` (WIP limit breach) or `422 Unprocessable Entity` (rule violation), the inline creation card must trigger a horizontal shake animation (`animate-shake` for 300ms) and display a detailed error banner (`data-testid="inline-card-error"`) inside the card.
*   **FR-55 (Submission Loading State)**:
    *   During execution of the POST request, the input textarea, `Add` button, and `Cancel` button must be marked as `disabled` and `aria-disabled="true"`. The `Add` button must render a white loading spinner.

## 🌐 2. REST API Contract: `POST /api/workspaces/:workspace_id/cards`

### 2.1 Route Mapping & Security
*   **Method**: `POST`
*   **Path**: `/api/workspaces/:workspace_id/cards`
*   **Security**: Authentication session validation via `WorkspaceTenantGuard` middleware. Returns `401 Unauthorized` if session is missing, and `403 Forbidden` if the user is not an active member of the specified `workspace_id`.

### 2.2 Request Payload JSON Schema
```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "title": "CreateCardRequest",
  "type": "object",
  "properties": {
    "title": {
      "type": "string",
      "minLength": 1,
      "maxLength": 255
    },
    "current_column_id": {
      "type": "string",
      "format": "uuid"
    },
    "current_swimlane_id": {
      "type": "string",
      "format": "uuid"
    }
  },
  "required": ["title", "current_column_id", "current_swimlane_id"],
  "additionalProperties": false
}
```

*   **Body Schema (JSON)**:

### 2.4 Error Responses
*   `400 Bad Request` (Validation failure): Title empty or too long.
*   `409 Conflict` (WIP Limit breach): Creation violates Column/Swimlane WIP constraints.
*   `422 Unprocessable Entity` (Arrival rule breach): Column arrival checklist prevents card creation in that column without required parameters.

## 💾 3. Database Integrations & Seeding DDL

To ensure absolute multi-tenant safety and relational integrity, card creation must execute inside a single PostgreSQL transaction (`BEGIN ... COMMIT`).

### 3.1 DDL Schema Consistency
Card creation maps to the existing tables (`cards` and `card_transitions`).

### 3.2 Transaction Boundary Execution
The database service tier in the backend must execute:
1.  **Pessimistic Read Locks**: Run a query checking active counts in the target column and swimlane using `FOR UPDATE` on their respective WIP records, guarding against concurrent overflow.
2.  **Insert Card**: Write the new row to the `cards` table.
3.  **Audit Event**: Write a `'create'` audit record to the `card_transitions` table.
4.  **Commit**: Commit the transaction. If any step fails, roll back completely.

## 🧪 4. TDD & Integration Test Blueprint

### 4.1 Rust Integration Test (`apps/api/tests/card_creation_api_tests.rs`)
Ensures validation, multi-tenancy, and WIP limits are validated under TDD:

```rust
#[sqlx::test]
async fn test_transactional_card_creation_and_wip_limits(pool: sqlx::PgPool) -> anyhow::Result<()> {
    sqlx::migrate!("./migrations").run(&pool).await?;

    let workspace_id = Uuid::new_v4();
    let col_id = Uuid::new_v4();
    let lane_id = Uuid::new_v4();
    let user_id = Uuid::new_v4();

    // 1. Seed base structures
    sqlx::query("INSERT INTO workspaces (id, name) VALUES ($1, 'Workspace Acme')").bind(workspace_id).execute(&pool).await?;
    sqlx::query("INSERT INTO users (id, email, name) VALUES ($1, 'user@kanbrio.io', 'User Admin')").bind(user_id).execute(&pool).await?;
    sqlx::query("INSERT INTO workspace_members (workspace_id, user_id, role) VALUES ($1, $2, 'admin')").bind(workspace_id).bind(user_id).execute(&pool).await?;

    // Set WIP limit of 1 on the column
    sqlx::query("INSERT INTO columns (id, workspace_id, title, position, wip_limit) VALUES ($1, $2, 'To Do', 1, 1)")
        .bind(col_id).bind(workspace_id).execute(&pool).await?;
    sqlx::query("INSERT INTO swimlanes (id, workspace_id, title, position) VALUES ($1, $2, 'Swimlane 1', 1)").bind(lane_id).bind(workspace_id).execute(&pool).await?;

    // 2. Create first card (Should succeed)
    let card1 = Card::create(
        &pool,
        CreateCard {
            parent_id: None,
            workspace_id,
            title: "Task 1".to_string(),
            current_column_id: col_id,
            current_swimlane_id: lane_id,
        }
    )
    .await?;
    assert_eq!(card1.title, "Task 1");

    // 3. Create second card in same column (Should fail due to WIP Limit of 1)
    let card2_res = Card::create(
        &pool,
        CreateCard {
            parent_id: None,
            workspace_id,
            title: "Task 2".to_string(),
            current_column_id: col_id,
            current_swimlane_id: lane_id,
        }
    )
    .await;

    assert!(card2_res.is_err());
    match card2_res.unwrap_err() {
        crate::AppError::WipLimitExceeded { entity, limit } => {
            assert_eq!(entity, "column");
            assert_eq!(limit, 1);
        }
        _ => panic!("Expected WipLimitExceeded error"),
    }

    Ok(())
}
```

### 4.2 Playwright E2E Integration Test Blueprint (`apps/e2e/tests/card_creation.spec.ts`)
Verifies interface focus trap, transitions, and inline validations:

```typescript
import { test, expect } from '@playwright/test';

test.describe('Card Creation Inline Form E2E', () => {
  test('should enforce inline form mount, input validations, loading states, and error shaking', async ({ page }) => {
    // 1. Authenticate and land on Board page
    await page.goto('/w/acme-workspace-id');

    // 2. Click "+ Add Card" in the first grid cell
    const addCardBtn = page.locator('[data-testid^="column-add-card-button-"]').first();
    await expect(addCardBtn).toBeVisible();
    await addCardBtn.click();

    // 3. Form mounted & programmatically focused
    const formCard = page.locator('[data-testid^="inline-card-form-"]').first();
    await expect(formCard).toBeVisible();
    const titleInput = formCard.locator('[data-testid="inline-card-title-input"]');
    await expect(titleInput).toBeFocused();

    // 4. Validate focus trap
    await page.keyboard.press('Tab'); // Add button
    await page.keyboard.press('Tab'); // Cancel button
    await page.keyboard.press('Tab'); // Loops back to title textarea
    await expect(titleInput).toBeFocused();

    // 5. Test Escape dismissal
    await page.keyboard.press('Escape');
    await expect(formCard).toBeHidden();
    await expect(addCardBtn).toBeFocused(); // Focus restored to trigger

    // 6. Reopen and test field validation (submit empty title)
    await addCardBtn.click();
    await page.keyboard.press('Enter');

    const errorMsg = formCard.locator('[data-testid="inline-card-error"]');
    await expect(errorMsg).toBeVisible();
    await expect(errorMsg).toContainText('Workspace name must contain at least 1 non-whitespace character.'); // or standard message
    await expect(formCard).toHaveClass(/animate-shake/);

    // 7. Successful creation
    await titleInput.fill('Deploy Auth Microservice');
    await formCard.locator('[data-testid="inline-card-submit"]').click();

    // Verification loading states
    await expect(titleInput).toBeDisabled();
    await expect(formCard).toBeHidden(); // closes on success

    // Assert card is loaded on board
    await expect(page.locator('text=Deploy Auth Microservice')).toBeVisible();
  });
});
```
